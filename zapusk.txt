Локальный запуск без секретов в файлах:

1. Заполнить `.env` реальными значениями только на машине/сервере.
2. Поднять Postgres:
   `docker compose up -d db`
3. Для ручного запуска приложения в PowerShell:
   `.\venv\Scripts\Activate.ps1`
   `alembic upgrade head`
   `python -m app.init_data`
   `python -m app.main`

Важно:
- `docker compose` в текущем виде поднимает только Postgres, само приложение запускается отдельно.
- Для локального запуска приложения с Postgres из `docker-compose.yml` обычно нужен `DATABASE_URL` с хостом `localhost` и портом `5433`, а не `db:5432`.
- Не хранить токены, пароли и приватные ключи в проектных файлах.
- Бот использует advisory lock в Postgres, поэтому polling стартует только в одном экземпляре.
- `python -m app.init_data` всегда заполняет тарифы и может создать/обновить сервер, если заданы `SEED_SERVER_*`.
